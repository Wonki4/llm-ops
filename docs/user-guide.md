## 시작하기

### 로그인

1. 포털 접속 시 SSO 로그인 페이지로 자동 이동됩니다.
2. 회사 계정 (사번)으로 로그인합니다.
3. 최초 로그인 시 자동으로 계정이 생성되고, 기본 팀에 배정됩니다.

### 사이드바 메뉴

로그인 후 왼쪽 사이드바에서 다음 메뉴를 사용할 수 있습니다:

| 메뉴 | 설명 |
|------|------|
| 모델 캘린더 | 모델 상태 변경 일정 확인 |
| 모델 대시보드 | 전체 모델 현황 조회 |
| 내 팀 | 소속 팀 관리 |
| 팀 탐색 | 새로운 팀 검색 및 가입 |
| 내 전체 키 | API 키 관리 |
| 내 요청 | 팀 가입/예산 변경 요청 현황 |

## 모델 캘린더

모델의 상태 변경 일정을 캘린더 형태로 한눈에 확인할 수 있습니다.

### 화면 구성

- **상단 통계**: 예정된 전환 수, 이번 달 이벤트, 폐기 예정 모델 수
- **캘린더**: 월별 그리드에 상태 전환이 색상 점으로 표시됩니다
- **예정 전환 목록**: 다가오는 상태 변경 20건을 테이블로 표시

### 사용 방법

1. **월 이동**: `<` / `>` 버튼으로 이전/다음 달 이동, "오늘" 버튼으로 현재 달 복귀
2. **상태 필터**: 드롭다운에서 특정 상태만 필터링 (Testing, Prerelease, LTS, Deprecating, Deprecated)
3. **긴급도 확인**: 예정 전환 목록에서 D-Day 카운트다운 확인
   - 빨간색: 7일 이내
   - 노란색: 30일 이내

### 상태 설명

| 상태 | 색상 | 의미 |
|------|------|------|
| Testing | 파란색 | 테스트 중인 모델 |
| Prerelease | 보라색 | 정식 출시 전 사전 공개 |
| LTS | 초록색 | 장기 지원 (Long Term Support) |
| Deprecating | 노란색 | 폐기 예정 (사용 주의) |
| Deprecated | 빨간색 | 폐기 완료 (사용 중단 권장) |

## 모델 대시보드

현재 사용 가능한 모델의 전체 현황을 조회합니다.

### 화면 구성

- **상단 통계 카드**: 전체 모델 수, 카탈로그 등록 수, 활성 모델 수, 폐기 예정 수
- **상태 분포**: 상태별 비율을 바 차트로 시각화
- **최근 변경**: 최근 5건의 모델 상태 변경 이력
- **모델 목록**: 전체 모델 테이블

### 사용 방법

1. **모델 검색**: 상단 검색창에서 모델명으로 검색
2. **상태 필터**: 드롭다운에서 특정 상태의 모델만 조회
3. **모델 상세 보기**: 모델명을 클릭하면 오른쪽에 상세 패널이 열립니다
   - 비용 정보 (Input/Output 비용)
   - 모델 제한 (Max Tokens, RPM, TPM)
   - 카탈로그 정보 (설명, 상태별 일정, 등록/수정일)
   - 상태 변경 이력

> 관리자가 숨김 처리한 모델이나 카탈로그에 미등록된 모델은 표시되지 않습니다.

## 내 팀

내가 소속된 팀 목록을 확인하고 관리합니다.

### 화면 구성

- 소속 팀이 카드 형태로 표시됩니다
- 각 카드에는 팀명과 사용 가능한 모델 수가 표시됩니다
- "모든 모델"로 표시되는 팀은 LiteLLM의 전체 모델에 접근 가능합니다

### 사용 방법

1. **팀 상세 보기**: 팀 카드를 클릭하면 팀 상세 페이지로 이동합니다
2. **팀이 없는 경우**: "팀 탐색하기" 링크를 통해 팀에 가입할 수 있습니다

---

## 팀 상세

팀의 상세 정보를 확인하고, 키/모델/예산을 관리합니다. 탭으로 구분됩니다.

### 개요 탭

팀의 전체 현황을 요약합니다.

- **통계 카드**: 팀 예산, 내 키 수, 모델 수, 내 예산
- **예산 현황**: 사용량/총량 프로그레스 바, 예산 주기, 초기화 일시

#### 예산 변경 요청

현재 예산을 변경하고 싶을 때:

1. 개요 탭 하단의 **"예산 변경 요청"** 버튼 클릭
2. 현재 예산이 표시됩니다
3. 변경하고자 하는 금액 입력 (변경 후 예산이 미리보기로 표시됩니다)
4. 사유 입력 (선택)
5. **"요청 제출"** 클릭
6. 관리자 승인 후 예산이 변경됩니다

### 내 키 탭

이 팀에서 내가 생성한 API 키를 관리합니다.

| 컬럼 | 설명 |
|------|------|
| 별칭 | 키 생성 시 설정한 이름 |
| 키 | 마스킹된 키 값 + 복사 버튼 |
| 만료일 | 키 만료 예정일 |
| 모델 | 사용 가능한 모델 수 |
| 생성일 | 키 생성 일시 |

#### 키 복사하기

1. 키 옆의 **복사 아이콘** 클릭
2. 실제 키 값이 클립보드에 복사됩니다
3. "키가 클립보드에 복사되었습니다" 알림 확인

#### 키 삭제하기

1. 해당 키의 **삭제 아이콘** 클릭
2. LiteLLM에서도 함께 삭제됩니다

### 모델 탭

이 팀에서 사용 가능한 모델 목록을 확인합니다.

- 카탈로그에 등록된 모델만 표시됩니다
- 모델명, 현재 상태, Input/Output 비용을 확인할 수 있습니다
- 모델명을 클릭하면 상세 정보를 볼 수 있습니다

## 팀 탐색

포털에 등록된 전체 팀을 검색하고 가입을 요청할 수 있습니다.

### 화면 구성

- 팀 카드: 팀명, 모델 수, 관리자 목록
- 상태 표시: 가입됨 / 요청중 / 가입 요청 버튼

### 팀 검색

1. 상단 검색창에서 팀명으로 검색
2. 필터 버튼으로 상태별 조회:
   - **전체**: 모든 팀
   - **가입됨**: 내가 소속된 팀
   - **미가입**: 아직 가입하지 않은 팀
   - **요청중**: 가입 요청이 대기 중인 팀

### 팀 가입 요청

1. 가입하려는 팀의 **"가입 요청"** 버튼 클릭
2. 가입 요청 모달에서 메시지 입력 (선택)
3. **"요청 보내기"** 클릭
4. 팀 관리자가 승인하면 자동으로 팀에 추가됩니다
5. 요청 현황은 **내 요청** 메뉴에서 확인할 수 있습니다

> 이미 가입된 팀이나 요청 중인 팀에는 중복 요청할 수 없습니다.

---

## 내 전체 키

모든 팀에서 생성한 API 키를 한곳에서 관리합니다.

### 화면 구성

- **통계 카드**: 전체 키 수, 소속 팀 수
- **키 테이블**: 별칭, 키 (마스킹 + 복사), 팀, 생성일, 삭제

### 키 검색 및 필터

1. **검색**: 별칭이나 키 값으로 검색
2. **팀 필터**: 드롭다운에서 특정 팀의 키만 조회

### 키 복사

1. 키 옆의 **복사 아이콘** 클릭
2. 실제 키 값이 클립보드에 복사됩니다

### 키 삭제

1. 삭제 아이콘 클릭
2. 확인 후 키가 영구 삭제됩니다 (LiteLLM에서도 함께 삭제)

## API 키 생성

새로운 API 키를 생성합니다.

### 생성 방법

1. **내 전체 키** 또는 **팀 상세 > 내 키 탭**에서 **"키 생성"** 클릭
2. **팀 선택** (필수): 키를 생성할 팀을 선택합니다
3. **키 별칭** (필수): 키를 구분하기 위한 이름을 입력합니다 (예: my-project-key)
4. 사용 가능 모델과 TPM/RPM 제한이 읽기 전용으로 표시됩니다
5. **"키 생성"** 클릭

### 키 확인 및 복사

1. 키가 성공적으로 생성되면 모달에 키 값이 표시됩니다
2. **복사 아이콘**을 클릭하여 클립보드에 복사합니다
3. **"확인"**을 클릭하면 이전 페이지로 돌아갑니다
4. 나중에 **내 전체 키**에서 언제든지 키를 다시 복사할 수 있습니다

### API 키 사용 방법

생성된 키를 LLM API 호출 시 사용합니다:

```bash
curl -X POST "https://{LiteLLM 주소}/v1/chat/completions" \
  -H "Authorization: Bearer {복사한 키}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "모델명",
    "messages": [{"role": "user", "content": "안녕하세요"}]
  }'
```

Python SDK 사용 시:

```python
from openai import OpenAI

client = OpenAI(
    api_key="{복사한 키}",
    base_url="https://{LiteLLM 주소}/v1"
)

response = client.chat.completions.create(
    model="모델명",
    messages=[{"role": "user", "content": "안녕하세요"}]
)
print(response.choices[0].message.content)
```

## 내 요청

내가 보낸 팀 가입 및 예산 변경 요청의 현황을 확인합니다.

### 화면 구성

- **검색**: 팀명으로 검색
- **상태 탭**: 전체 / 대기중 / 승인 / 거절
- **요청 테이블**: 유형, 팀, 내용, 상태, 요청일, 처리 코멘트

### 요청 상태

| 상태 | 의미 |
|------|------|
| 대기중 | 관리자 검토 대기 |
| 승인 | 요청이 승인되어 처리 완료 |
| 거절 | 요청이 거절됨 (사유는 처리 코멘트 참조) |

### 상세 확인

- 내용이나 처리 코멘트 셀을 클릭하면 전체 내용을 모달로 확인할 수 있습니다

## 자주 묻는 질문

### 로그인이 안 됩니다
- 회사 SSO 계정으로 로그인해야 합니다
- 계정에 문제가 있으면 IT 관리자에게 문의하세요

### 팀이 보이지 않습니다
- 관리자가 숨김 처리한 팀은 표시되지 않습니다
- 팀 탐색 메뉴에서 가입 가능한 팀을 확인하세요

### 모델이 보이지 않습니다
- 카탈로그에 등록되지 않았거나 숨김 처리된 모델은 표시되지 않습니다
- 사용하려는 모델이 보이지 않으면 관리자에게 문의하세요

### 키를 분실했습니다
- **내 전체 키** 메뉴에서 언제든지 키를 다시 복사할 수 있습니다
- 보안상 키를 재발급하려면 기존 키를 삭제하고 새로 생성하세요

### 예산이 부족합니다
- 팀 상세 > 개요 탭에서 **"예산 변경 요청"**을 통해 예산 증액을 요청할 수 있습니다
- 관리자 승인 후 예산이 반영됩니다

### 팀 가입 요청이 거절되었습니다
- **내 요청** 메뉴에서 처리 코멘트를 확인하세요
- 필요하면 다시 요청하거나 팀 관리자에게 직접 문의하세요

### API 호출 시 오류가 발생합니다
- 키가 올바른지 확인하세요 (내 전체 키에서 재복사)
- 모델명이 정확한지 확인하세요 (모델 대시보드에서 확인)
- TPM/RPM 제한에 도달했을 수 있습니다. 잠시 후 다시 시도하세요
- 예산이 소진되었을 수 있습니다. 예산 현황을 확인하세요
